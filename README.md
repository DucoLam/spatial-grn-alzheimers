# Spatially Annotated Gene Regulatory Networks in Alzheimer's Disease

**Author:** D. Lam вЂ” BSc Thesis, [TU Delft (CSE3000 Research Project)](https://github.com/TU-Delft-CSE/Research-Project), June 2026

**Compute:** DAIC HPC cluster (`login.daic.tudelft.nl`, SLURM + Apptainer)

Pipeline for inferring cell-specific gene regulatory networks (GRNs) with
**ScReNI**, projecting them onto MERFISH spatial coordinates with **Tangram**, and
analysing how GRN structure varies across tissue space and AD severity.

## Research question

> Can changes in the spatial location of cells in human Alzheimer's disease tissue
> be related to changes in gene regulatory network structure and the activity of
> key regulators?

- **SQ1** вЂ” Can cell-specific GRNs be reliably inferred (ScReNI) for the selected AD samples?
- **SQ2** вЂ” Do differences in GRN structure relate to differences in spatial location?
- **SQ3** вЂ” Do spatial differences in GRN structure relate to AD severity (CPS)?
- **SQ4** вЂ” Which specific regulators drive spatial GRN variation?

Analyses are run per cell subtype: **Astrocyte, L2/3 IT, L4 IT, Oligodendrocyte**.

## Project Scope & Experimental History
This repository contains the complete computational workflow for the thesis. To maintain transparency regarding the research process, please note the following:

* **Experimental Scripts:** The `analysis/` and `archive/` directories contain various exploratory scripts and intermediate analysis iterations. While the core pipeline is finalized, these files are preserved to provide a complete provenance of the methodology and the evolution of the results.
* **SpaGE Imputation:** The main pipeline includes modular steps for **SpaGE**-based gene expression imputation. Please be aware that while SpaGE was tested as a method to bridge gene-panel discrepancies, these specific outputs were **not included in the final results** of this thesis. The final GRNs and spatial projections presented in the paper rely solely on the primary ScReNI and Tangram integration.

## Running the pipeline

All scripts are submitted from the repo root with `sbatch jobs/<script>.sh`.
Spatial inference (Tangram, per subtype) uses `container.sif`; GRN inference
(ScReNI, pooled across the 4 subtypes) uses `container_0-1-3.sif`.

Pooled GRN chain:

```bash
J1=$(sbatch jobs/08_pool.sh | awk '{print $4}')
J2=$(sbatch --dependency=afterok:$J1 jobs/09_gene_peak_pool.sh pool4 | awk '{print $4}')
sbatch --dependency=afterok:$J2 jobs/10_wscreni_shard.sh pool4      # 16-shard array
sbatch jobs/13_spatial_link.sh                                       # join GRNs + coords
```

SQ analyses: `sbatch jobs/run_sq2_analysis.sh`, `jobs/run_sq3.sh`,
`jobs/run_sq4.sh`, etc. Outputs land in `analysis/SQ{1,2,3,4}/`.

## Data

Code, figures (`plots/`, `section_plots/`), and small derived result tables
(`analysis/**/*.csv`, `data/**/*.csv`) are tracked here. The **heavy binary
artifacts (~119 GB) are not on GitHub** and remain on the cluster umbrella mount
under `data/`:

- `data/wscreni_networks_pool4/` вЂ” 73,956 per-cell GRNs (~111 GB)
- `data/spatial_link/spatial_grn_matrix.npz` вЂ” cellГ—edge matrix (~3.6 GB)
- `data/*.h5ad`, `data/*.npz` вЂ” pooled RNA/ATAC, Tangram outputs, peak matrices

These are regenerated by running the pipeline above (they are deterministic;
`seed = 42` is used for all randomised computations).

## Reproducibility

The full pipeline runs inside the versioned Apptainer containers above (package
versions pinned via Pixi / `pixi.lock` in `bsc-screni`). All randomised steps use
a fixed `seed = 42`.
